Zappy Documentation

Un jeu de simulation multi-joueurs inspiré de Zaphod Beeblebrox

Qu'est-ce que Zappy?

Zappy est un jeu de simulation réseau où plusieurs équipes s'affrontent sur une carte en tuiles contenant des ressources. L'objectif est d'être la première équipe où au moins 6 joueurs atteignent le niveau d'élévation maximum.

Installer Guide d'utilisation

Fonctionnalités principales

Jeu en réseau

Communication client-serveur via TCP/IP

Interface graphique

Visualisation 2D/3D du monde avec SFML

IA autonome

Intelligence artificielle pour contrôler les joueurs

Système de ressources

6 types de pierres précieuses et nourriture

Installation

Prérequis

  • GCC (version 8.0 ou supérieure)
  • G++ (version 8.0 ou supérieure)
  • Make
  • SFML (pour l'interface graphique)
  • Python 3.8+ (pour l'IA)

Compilation

Terminal 
make

Cette commande va compiler les trois composants principaux :

  • zappy_server - Le serveur de jeu
  • zappy_gui - L'interface graphique
  • zappy_ai - L'intelligence artificielle

Utilisation

Démarrage du serveur

Serveur 
./zappy_server -p port -x width -y height -n name1 name2 ... -c clientsNb -f freq

Lancement de l'interface graphique

GUI 
./zappy_gui -p port -h machine

Démarrage d'un client IA

IA 
./zappy_ai -p port -n name -h machine

Serveur

Le serveur Zappy est écrit en C et gère le monde du jeu, les connexions clients et la logique de jeu.

Architecture du serveur

Gestion réseau

Communication TCP/IP avec multiplexage via poll()

Gestion équipes

Authentification et gestion des équipes

Monde de jeu

Génération et gestion du monde Trantor

Rituels d'élévation

Gestion des incantations et montées de niveau

Options du serveur

Options 
-p port      : Numéro de port
-x width     : Largeur du monde
-y height    : Hauteur du monde
-n name1...  : Noms des équipes
-c clientsNb : Nombre de clients autorisés par équipe
-f freq      : Fréquence d'exécution des actions

Interface Graphique

L'interface graphique est développée en C++ avec SFML pour la visualisation 2D du monde de jeu.

Fonctionnalités de l'interface

  • Visualisation en temps réel du monde
  • Affichage des joueurs et de leurs mouvements
  • Visualisation des ressources sur la carte
  • Interface utilisateur pour contrôler la vue
  • Animations des actions et rituels

Protocole de communication

Le protocole de communication définit les interactions entre le serveur et les clients.

Commandes principales

Commandes client 
Forward      : Avancer d'une case (7/f temps)
Right        : Tourner à droite (7/f temps)
Left         : Tourner à gauche (7/f temps)
Look         : Regarder autour (7/f temps)
Inventory    : Voir l'inventaire (1/f temps)
Broadcast    : Diffuser un message (7/f temps)
Connect_nbr  : Nombre de connexions disponibles
Fork         : Reproduire un joueur (42/f temps)
Eject        : Éjecter les joueurs (7/f temps)
Take object  : Prendre un objet (7/f temps)
Set object   : Poser un objet (7/f temps)
Incantation  : Commencer un rituel (300/f temps)

Séquence de connexion

Protocole de connexion 
1. Client se connecte au serveur
   <-- WELCOME\n

2. Client envoie le nom d'équipe
   --> TEAM-NAME\n

3. Serveur répond avec les informations
   <-- CLIENT-NUM\n
   <-- X Y\n

Exemples d'utilisation

Exemple complet de partie

1. Démarrer le serveur

Terminal 1 
./zappy_server -p 4242 -x 20 -y 20 -n team1 team2 team3 -c 4 -f 100

2. Lancer l'interface graphique

Terminal 2 
./zappy_gui -p 4242 -h localhost

3. Connecter des clients IA

Terminal 3-N 
./zappy_ai -p 4242 -n team1 -h localhost
./zappy_ai -p 4242 -n team2 -h localhost
./zappy_ai -p 4242 -n team3 -h localhost
